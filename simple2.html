#include <iostream>

using namespace std;
int i=1;
main()
{
	cout << "i=" << i << endl;
	int i=2;
	cout << "i=" << i << endl;

	{//inner block
	int i=3;
	cout <<"  i=" << i << endl;
	cout <<"  i=" << i << endl;
	}//inner block
	cout << "i=" << i << endl;
	{//inner block 2
	int i=4;
	cout <<"  i=" << i << endl;
	}//inner block 2

	cout << "i=" << i << endl;
}//end main()

  simple2.cpp
  There are some interesting things about how scoping affects the program execution.

  + There is a global variable 'int i=1;'. When you print out 'i' in main but no local variables 'i' are declared yet, the program prints 1.
  + After declaring the local variable 'int i=2;' When you print out 'i' again the program prints 2 because local variables have higher precedences over global variables by default.
  + When we do some scoping (inner block), we can override previously declared local variables that initially lie outside the inner block.
  There we have "int i=3;" which temporarily overrides the local variable 'int i=2;' outside the inner block.
  When we print 'i' two times, the program chooses the most recent instance of 'i' local variable. (which is "int i=3;")
  Note that you have to do scoping to override a previously declared local variable, otherwise it will result in a compiler error (variable redefinition).

  + When the inner block ends and you print out 'i' again, the result is "int i=2;".
  The inner block ends and it can be said that all variables declared in the block nolonger exist.
  Meaning that only local variables declared in the scope exist within the scope.
  When the program finishes executing the code in a scope we nolonger consider the existence of the scope and all local variables within.
  Therefore "int i=3;" no longer exists, so program chooses the most recent instance of 'i' local variable (which is 'int i=2;').

  + We can do some scoping to temporarily override previously declared (local or global variable) outside the scope.
  Meaning that we can again declare another 'i' variable in the inner block 2 ("int i=4;")
  When we print 'i', the program chooses the most recent instance of 'i' local variable. (which is "int i=4;")
  When the inner block 2 ends, we can safely say the most recently declared 'i' variable "int i=4;" is destroyed since the local variable lies within inner block 2.
  When we print out 'i' again, the first local variable 'int i=2;' is used. The program already finished executing the inner block 2 so we will not consider any local variables within it.
  Since 'int i=2;' is a general local variable which does not belong to any particular local scopes the variable lives for the rest of the function.
